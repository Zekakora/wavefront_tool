# Wavefront Tool (波前分析与信号处理工具)

## 1. 简介 (Introduction)
Wavefront Tool 是一个基于 Python 和 PyQt 构建的桌面端图形化信号分析软件。该工具专注于复杂信号的**波前检测 (Wavefront Detection)** 和**初至时间拾取 (Arrival Time Picking)**，并集成了小波变换等时频分析功能。

项目内置了多种先进的信号处理算法（如 ICEEMDAN-TEO 联合算法、RDP-AIC 算法等），旨在为声发射信号、超声检测、地震波信号等领域的科研工作者和工程师提供一个可视化、参数可调且易于操作的分析平台。

## 2. 主要功能 (Key Features)

* **多算法波前检测引擎**:
  * **RDP-AIC 算法** (`wavefront_algo_rdp_aic.py`): 结合 Ramer-Douglas-Peucker (RDP) 曲线压缩与 Akaike Information Criterion (AIC) 实现高效的局部与全局波前拾取。
  * **ICEEMDAN-TEO 算法** (`wavefront_algo_iceemdan_teo.py`): 基于改进的自适应噪声完备集合经验模态分解 (ICEEMDAN) 与 Teager 能量算子 (TEO) 的复杂信号包络与特征提取。
* **高级时频分析**: 提供专门的小波变换分析面板 (`wavelet_transform_panel.py`)，支持信号的多尺度时域与频域洞察。
* **交互式参数配置 GUI**: 动态生成参数调整界面 (`wavefront_param_dialog.py`)，并配备完善的参数状态存储与数据校验 (`wavefront_param_schema.py`, `wavefront_param_store.py`)。
* **数据读写与可视化**:
  * 灵活的数据导入与结果导出模块 (`wavefront_data_io.py`)。
  * 交互式图表绘制及高质量图像保存功能 (`wavefront_plot_save.py`)。
* **本地化**: 支持多语言界面映射 (`wavefront_param_translations.py`)。

## 4. 核心算法简介

系统内置了两套互补的波头检测算法，可根据信号质量与检测需求在界面中进行切换：

### 4.1 RDP + 局部 AIC 算法 (`wavefront_algo_rdp_aic.py`)
* **特点**: 运行速度极快，适合处理具有明显突变且信噪比中等的快速行波信号。
* **原理**: 首先利用 Ramer-Douglas-Peucker (RDP) 算法对平滑后的信号进行几何抽稀，快速定位异常转折点。随后在转折点附近展开双门限（幅值与斜率）精细搜索，最后通过 Akaike Information Criterion (AIC) 寻找方差突变的极小值点以实现统计学精确定位。

### 4.2 全局 ICEEMDAN-TEO 算法 (`wavefront_algo_iceemdan_teo.py`)
* **特点**: 抗干扰能力极强，专门应对强背景噪声、低信噪比以及起步平缓的高难度信号，但计算耗时相对较长。
* **原理**: 利用改进的自适应噪声完备集合经验模态分解 (ICEEMDAN) 将信号分解为多个固有模态函数 (IMF)，有效滤除高低频环境噪声。随后计算选定 IMF 的 Teager 能量算子 (TEO)，通过捕捉瞬态能量的突变来实现高精度的波前识别。

# 参数说明与调参指南

本工具内置了两套波头识别算法：**RDP + 局部 AIC 算法** 与 **全局 ICEEMDAN-TEO 算法**。波头检测往往需要在“敏感度（捕获微弱波头）”与“抗干扰（过滤背景噪声）”之间寻找平衡。以下是所有核心参数的详细说明与调参建议。

---

## 一、 通用与数据截取参数 (General Parameters)

* **`fs` (采样频率)**
    * **作用**：信号的硬件采样频率（Hz）。
    * **影响**：算法内部所有基于“时间窗 (us)”的参数，都会通过 `fs` 换算为“样本点数”。设置错误会导致所有时间窗长度全部失效。
* **`pre_n` / `rdp_pre_n` (前置基线样本数)**
    * **作用**：评估背景环境噪声的初始纯净样本数。算法利用这段数据计算“零点基线”和“噪声标准差”。
    * **影响**：**这段数据内绝对不能包含真正的波头！** 如果设置太长导致包含了波头，基线噪声标准差（Sigma）会被严重拉高，导致后续所有基于 K 倍阈值的判断失效（即漏检）。
* **`polarity` (极性)**
    * **作用**：波头的极性方向。可选 `auto`(自动判定)、`positive`(正向突破)、`negative`(负向突破)。

---

## 二、 RDP + 局部 AIC 算法参数

此算法适合处理具有明显突变、信噪比中等的快速行波信号。处理速度极快。

### 1. 小波降噪阶段 (Wavelet Denoising)
* **`wavelet_enabled` (启用小波降噪)**: 是否在特征提取前去除高频白噪声。
* **`wavelet_level` (小波分解层数)**: 默认 4 层。
    * **影响**：层数越高，滤除的低频背景干扰越多。**警告**：层数过高可能导致真实的波头前沿被“磨平”，导致起步时间滞后。
* **`wavelet_beta` & `wavelet_threshold_scale`**: 控制降噪软阈值函数的宽容度。`scale` 越大，降噪力度越强，抹除的细节越多。

### 2. RDP 抽稀与粗定位阶段 (Rough Locator & RDP)
算法会先找一个大概的突变点，然后再精细搜索。
* **`rough_k` (粗定位阈值 K)**
    * **作用**：粗定位寻找偏离基线超过“K倍噪声水平”的点。
    * **影响**：设得太大容易漏检微弱波头；设得太小容易被环境毛刺触发。
* **`rough_consecutive` (粗定位连续点数)**
    * **作用**：信号必须连续越过粗定位阈值的点数。
    * **影响**：增大此值能极好地过滤单点或双点的尖峰放电干扰。
* **`rdp_epsilon` (RDP 抽稀容差)**
    * **作用**：几何抽稀的垂直容差限制。
    * **影响**：值越大，忽略的细节波动越多，提取出的几何转折点越少；值过小会导致保留大量无用的噪声毛刺，失去 RDP 的提纯意义。

### 3. 双门限精细搜索阶段 (Fine Search)
在粗定位点附近展开双门限（幅值 + 斜率）搜索。
* **`search_left_us` / `search_right_us` (精搜左/右窗口)**
    * **作用**：在粗定位点向左/右扩展搜索的时间范围（微秒）。
    * **调参**：如果发现粗定位总是比真正的波头晚，需要增大 `search_left_us`。
* **`amp_k` (幅值阈值 K)** & **`slope_k` (斜率阈值 K)**
    * **作用**：精定位判定波头的双重门限。
    * **影响**：调低这两个值，波头时间会明显“提前”（更早发现微小上扬）；调高则抗噪性更强，但容易晚拾取。
* **`min_consecutive` (精搜最少连续点数)**
    * **作用**：幅值和斜率必须**同时且连续**达标的最少点数。防范毛刺误触发的核心参数。

### 4. AIC 统计精确定位阶段 (Local AIC)
* **`aic_left_us` / `aic_right_us`**: AIC 函数计算方差的局部数据窗。AIC 会在该窗口内寻找方差突变的全局极小值点作为最终波头。
* **`aic_smooth_win` (AIC 平滑窗口)**: 平滑 AIC 离散曲线以寻找唯一的极小值，避免多极小值震荡。

---

## 三、 ICEEMDAN-TEO 算法参数

此算法抗噪能力极强，专门应对强背景噪声、低信噪比、缓波头等极高难度的信号，但计算耗时较长。

### 1. ICEEMDAN 模态分解阶段 (Decomposition)
* **`ensemble_size` (集合数)**: 默认 50。
    * **影响**：向原信号中加入白噪声的次数。**直接决定算法运行时间。** 如果只需粗略结果，可降至 20-30 以提速；要求极高稳定性时设为 50-100。
* **`noise_strength` (噪声强度)**: 默认 0.10。解决模态混叠问题。通常保持默认。
* **`use_mirror_pad` & `mirror_pad_us` (镜像填充)**
    * **作用**：对信号两端进行镜像延拓后再分解。
    * **影响**：极大地减弱了经验模态分解中非常棘手的“端点发散效应”（飞线）。**强烈建议保持开启。**

### 2. TEO 能量算子阶段 (Teager Energy Operator)
* **`use_imf_mode` (IMF 使用方式)**: 可选 `iimf1` 或 `iimf12`。
    * **调参**：行波波头通常是极高频信号，理论上 `iimf1` 就足够。但如果是经过长距离衰减的“缓波头”，高频分量流失，请选择 `iimf12`（融合第一和第二模态），并通过增大 **`alpha2` (IMF2权重)** 来捕捉缓变前沿。
* **`sigma_k` (TEO 能量Sigma阈值 K)**
    * **作用**：寻找 TEO 能量曲线上超过 K 倍背景能量的点。
    * **影响**：TEO 会放大概率突变，`sigma_k` 通常可以设得比较高（如 4.0 ~ 7.0）。

### 3. 波头拾取阶段 (Picking Mode)
* **`pick_mode` (拾取模式)**:
    * `first_cross`: 首次连续越阈。**最推荐，最贴近物理波头起步点。**
    * `max`: 搜索窗内能量最大峰。通常位于波头起步后的半个周期，会有固定滞后。
    * `first_sig_slope`: 首个显著斜率突变点。
* **`cross_consecutive` (越阈连续点数)**: 搭配 `first_cross` 使用，防止 TEO 能量尖峰导致误触发。

## 四、 调参指南 (Troubleshooting Guide)

在实际应用中，您可能会遇到以下几种典型情况：

### 症状 1：频繁将噪声误识别为波头（过敏感）
* **现象**：程序找出的波头在真正的波头之前，抓在了某个毛刺上。
* **对策**：
    1.  **首选**：增加 `min_consecutive`（RDP算法）或 `cross_consecutive`（TEO算法）的值（例如从 3 提高到 5 甚至 8）。要求干扰信号必须持续足够长的时间，直接秒杀短促毛刺。
    2.  **次选**：适当提高 `rough_k`（粗定位门限）或 `amp_k` / `sigma_k`。
    3.  **如果毛刺极多**：确保开启了 `wavelet_enabled`，或者稍微调大 `rdp_epsilon` 强行拉平背景波动。

### 症状 2：漏检，找不到明显的微弱波头（不敏感）
* **现象**：波头很平缓或者幅度很小，算法直接报错找不到，或者定位到了后面的大反射波上。
* **对策**：
    1.  **检查 `pre_n`**：确保 `pre_n` 框定的基线样本中**完全没有波头信号**。如果有，算法会认为目前的波头也是“正常噪声”，从而大幅提高阈值导致漏检。
    2.  **降低门限**：调小 `rough_k`、`amp_k` 或 `slope_k`。
    3.  降低 `wavelet_level` 或关闭小波降噪，以防真实的微弱高频突变被滤除。

### 症状 3：波头定位点总是“滞后”（爬坡晚）
* **现象**：肉眼看信号已经开始上扬了，但算法标记的红线却晚了几个微秒。
* **对策**：
    1.  增大精搜左窗口 `search_left_us` / `aic_left_us`，让算法向左（历史时间）回溯得更远。
    2.  降低斜率阈值 `slope_k`，因为早期的上扬斜率通常很小。
    3.  如果是 ICEEMDAN 算法，尝试使用 `iimf12` 模式并提高 `alpha2`，让低频分量参与计算。

### 症状 4：处理速度太慢，界面卡顿
* **对策**（主要针对 ICEEMDAN-TEO）：
    1.  必须开启 `rdp_preprocess`！这能让模态分解只在局部发生，而不是对几十万个点进行全量分解。
    2.  调小 `ensemble_size`，从 50 降至 20 甚至 10（牺牲极小部分稳定性换取数倍提速）。
    3.  限制 `max_imfs` 为 4 或 5。行波能量基本都在前 2 个模态，后续分解完全是浪费时间。
